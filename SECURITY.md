# Security Policy

## Reporting a Vulnerability

Please report security vulnerabilities responsibly by emailing **x@algotyrnt.com** instead of posting them to public issue trackers.

When reporting a vulnerability, please include:
- Description of the vulnerability
- Steps to reproduce it (if applicable)
- Potential impact
- Suggested fix (if you have one)

## Security Considerations

This is a personal portfolio built with:
- **Next.js 16** for the frontend framework
- **TypeScript** for type safety
- **Material-UI** for component library
- **ESLint & Prettier** for code quality

### Dependencies

We regularly update our dependencies to incorporate security patches. If you discover a security issue in one of our dependencies:

1. Check if an updated version is available
2. Update via `npm install` to the latest secure version
3. Report critical vulnerabilities to us directly
